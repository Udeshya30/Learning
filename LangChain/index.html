1. What is LangChain?

LangChain is an orchestration framework for building applications powered by Large Language Models (LLMs).

Main Purpose

  • Connect LLMs to external tools
  • Build RAG pipelines
  • Create AI agents
  • Manage prompts
  • Handle memory
  • Orchestrate workflows
LangChain is not the AI model itself. It is the orchestration layer around AI systems.

2. Why LangChain Exists

Raw LLM APIs are not enough for production AI systems.

Without LangChain With LangChain
Manual orchestration Structured pipelines
Custom integrations Unified abstractions
Complex memory handling Built-in memory systems
Ad-hoc prompting Reusable prompt templates

7. Chains

Chains connect multiple operations together. 

from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template(
    \"Explain {topic}\"
)

chain = prompt | llm | StrOutputParser()

result = chain.invoke({
    \"topic\": \"LangChain\"
})
Modern LangChain uses LCEL (LangChain Expression Language).

8. LCEL (LangChain Expression Language)

LCEL is the modern pipeline syntax in LangChain. 

chain = prompt | llm | parser

Benefits

  • Composable pipelines
  • Streaming support
  • Parallel execution
  • Cleaner architecture
LCEL replaced many older chain abstractions.

16. Building a RAG Pipeline

retrieved_docs = retriever.invoke(query)

context = \"\\n\".join(
    [doc.page_content for doc in retrieved_docs]
)

prompt = f\"\"\"
Use this context:

{context}

Answer:
{query}
\"\"\"

RAG Flow

  1. User query
  2. Generate embeddings
  3. Vector search
  4. Retrieve context
  5. Inject into prompt
  6. Generate grounded answer
